# AI in Education — The State of the Art

**A Living Book in One Hundred Questions**

## What is this?

A "Living Book" whose structure is a fixed set of 100 questions about AI in education, and whose content (the answers) is automatically regenerated by an LLM with web search at scheduled intervals. When the state of knowledge changes, the book updates itself. Human intervention is limited to reviewing a changelog.

## How it works

1. **Question bank** (`questions/questions.json`) — 100 questions organized in 12 Parts (I–XII), covering philosophical foundations, ethics, teaching practice, learner experience, institutions, economy, evidence, field experiences, policy, practical navigation, measurement, and frontiers.

2. **Scheduled update pipeline** (`scripts/update.js`) — A GitHub Actions workflow runs weekly, sends each question to the Anthropic API (Claude) with web search enabled, and retrieves a current answer with cited sources.

3. **Change detector** — Before overwriting any stored answer, the system asks the LLM to compare old vs. new and classify the difference as:
   - **(A)** No substantive change — old answer stays
   - **(B)** Updated facts — new answer replaces old, logged in changelog
   - **(C)** Conclusion reversed — flagged for human review before publication

4. **Changelog** (`changelog/changelog.json`) — A human-readable record of every content change, published as an appendix ("the book's intellectual history").

5. **Automated assembly** (`scripts/build.js`) — Regenerates the full book as a static website (`docs/index.html`) served by GitHub Pages.

6. **Coherence layer** — A persistent voice prompt in `book.config.json` applied to all answer generation keeps 100 separately-written essays sounding like one author.

7. **Safeguards** — Source citation required for every factual claim; changes must be justified; conclusion reversals quarantined for human review; failures never corrupt the existing book.

## Repository structure

```
living-book/
├── book.config.json          # Title, model, voice prompt, settings
├── questions/
│   └── questions.json        # The 100-question bank (the only hand-authored file)
├── answers/                  # One JSON file per question (pipeline-generated)
├── changelog/
│   └── changelog.json        # Machine-readable change log
├── review/
│   └── pending.json          # Conclusion reversals awaiting human review
├── scripts/
│   ├── prompts.js            # The 3 LLM prompts (answering, change detection, voice)
│   ├── update.js             # The update pipeline
│   └── build.js              # Website assembly script
├── docs/                     # Generated book website (GitHub Pages serves this)
├── .github/workflows/
│   └── update.yml            # GitHub Actions weekly schedule
├── prepublish_check.py       # Validation: checks Q1–Q100 heads, regenerates TOC,
│                             #   validates cross-references, builds reverse index
├── package.json              # Node.js manifest
└── README.md                 # This file
```

## Running manually

```bash
# Set your API key
export ANTHROPIC_API_KEY="sk-ant-..."

# Run the update pipeline
npm run update

# Build the website
npm run build
```

## Costs

- **Per question per run:** ~$0.06–0.10
- **Full refresh (100 questions):** ~$6–10
- **Weekly schedule:** ~$25–45/month
- **Monthly schedule:** ~$6–10/month

GitHub repository, Actions minutes (public repos), and Pages hosting cost $0.

## Monthly maintenance ritual

1. Read the changelog appendix — check that justifications describe real changes in knowledge.
2. Open `review/pending.json` — approve or reject any quarantined conclusion reversals.
3. Glance at the edition stamp on the public page — it should never be older than one cycle.

## Requirements

- Node.js 20+
- Anthropic API key (stored as GitHub secret `ANTHROPIC_API_KEY`)
- GitHub repository (public, for free Actions and Pages)
